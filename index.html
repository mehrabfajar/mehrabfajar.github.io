Process

How I Work

A few principles that guide every project I take on — from a quick prototype to a full production system.

Data before models

The best algorithm can't fix bad data. I spend disproportionate time understanding data provenance, distributions, and failure modes before writing a single model line.

Ship something dumb first

A working linear baseline beats a stalled deep learning experiment every time. I establish a floor fast, then iterate. Complexity earns its place — it isn't assumed.

Evaluation is the product

I treat the evaluation harness as a first-class deliverable. If you can't measure it rigorously, you can't improve it reliably. Leaderboard metrics rarely tell the full story.

Explainability is respect

Every stakeholder deserves to understand what the model is doing and why. I default to interpretable systems, and add complexity only where the tradeoff is explicit.
